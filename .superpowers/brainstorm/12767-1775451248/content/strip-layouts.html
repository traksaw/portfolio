C — Asymmetric Feature

Writing post gets a hero-sized card with summary; talk and code are smaller side items. Emphasizes your latest writing as the main draw.

Key differences

All approaches share

  • Items gracefully hide when empty (e.g., no talks yet)
  • Auto-fetched data — RSS for writing, GitHub API for repos
  • Server-side with revalidation (no client JS needed)
  • Uses existing th-* design tokens

Trade-offs

  • A — Most visual weight, takes up the most vertical space
  • B — Least visual weight, but may feel too minimal between the identity cards and about section
  • C — Best if writing is your primary content; awkward if there's only 1 side item
